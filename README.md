## 安装方式

### 方式一：使用安装包（推荐）

1. 下载最新的 `Electron Paper-Setup.exe` 安装包
2. 运行安装程序，选择您想要的安装目录
3. 应用将安装到您选择的位置并创建开始菜单快捷方式
4. 从开始菜单或桌面快捷方式启动应用

### 方式二：使用便携版

1. 下载便携版应用文件夹
2. 解压到任意位置
3. 运行目录中的 `Electron Paper.exe` 启动应用

### 方式三：开发环境运行

确保您已安装 [Node.js](https://nodejs.org/)。

```bash
# 克隆项目
git clone https://github.com/yourusername/electron-paper.git

# 进入项目目录
cd electron-paper

# 安装依赖
npm install

# 开发模式启动（支持热重载）
npm run dev

# 或直接启动
npm start
```

## 使用说明

1. **搜索论文**：
   - 在搜索框中输入关键词或arXiv ID（如"2502.14776"）
   - 点击搜索按钮或按回车键开始搜索
   - 支持多条件组合搜索
   - 可设置每页显示结果数量
   - 搜索结果将显示在界面中

2. **查看论文详情**：
   - 点击搜索结果中的论文标题展开详细信息
   - 详情包括完整摘要、作者列表、发布日期和分类
   - 支持查看和访问论文分类

3. **使用AI助手**：
   - 点击界面右上角的"召唤Awei"按钮打开对话框
   - 通过文本输入框输入您的学术问题或需求
   - AI助手将基于当前上下文和SiliconFlow LLM生成回答
   - 学术问题回复会自动包含分析部分和关键词提取
   - 提供快捷检索选项，帮助快速找到相关论文
   - 支持代码块和Markdown格式的显示
   - 语音输入功能正在开发中，交互组件已部署

4. **翻译功能**：
   - 在设置中配置并保存SiliconFlow API密钥
   - 点击"翻译"按钮将论文标题和摘要翻译成中文
   - 点击"显示原文"可以切换回英文原文
   - 支持批量翻译多篇论文

5. **知识库管理**：
   - 将感兴趣的论文添加到知识库
   - 添加个人笔记和阅读状态
   - 支持中英文双语存储和切换
   - 卡片式布局直观管理论文
   - 导出为JSON或转换为Excel

6. **下载和访问**：
   - 点击"PDF"按钮直接下载或在浏览器中打开论文PDF
   - 点击"arXiv"按钮访问论文在arXiv的官方页面
   - 批量选择和导出多篇论文

7. **批量操作**：
   - 使用顶部复选框选择/取消选择所有论文
   - 单独选择需要的论文
   - 批量导出所选论文到知识库或JSON文件

## 翻译功能与AI助手配置

1. 访问 [SiliconFlow](https://cloud.siliconflow.cn/account/ak) 获取API密钥
2. 在应用的"设置"选项卡中填入API密钥
3. 选择合适的翻译模型和AI助手模型
4. 保存设置后即可使用翻译功能和AI助手功能

## 打包说明

本项目提供打包脚本，可以轻松创建便携版和安装版应用。

### 一键打包（推荐）

使用打包脚本可同时生成便携版和安装版：

```bash
# 以管理员身份运行打包脚本
.\package.bat
```

此脚本会自动：
1. 结束可能运行的应用程序进程
2. 清理旧版本和已安装的应用
3. 安装或更新必要的依赖
4. 直接使用electron-builder创建便携版应用和自定义安装程序
5. **自动包含python_env虚拟环境目录，解压或安装后可直接使用，无需手动配置Python环境**

### 手动打包

如果需要单独执行打包步骤：

```bash
# 创建便携版和支持自定义安装路径的NSIS安装程序
npm run build-nsis
```

此命令会：
1. 清理旧版本和已安装的应用
2. 使用electron-builder直接创建便携版应用和Windows安装程序
3. **自动包含python_env虚拟环境目录**

输出：
- 便携版：`dist/win-unpacked/` (包含 `Electron Paper.exe` 和 `python_env` 目录)
- 安装程序：`dist/Electron Paper Setup 1.0.0.exe`

### 单独打包便携版

如果只需要便携版，可以使用：

```bash
# 仅打包便携版
npm run dist:simple
```

> **注意：**
> - 打包包体已自动包含完整的 `python_env` 虚拟环境，用户解压或安装后可直接使用，无需手动配置Python环境。
> - 若需更换或升级Python依赖，请在 `python_env` 目录中操作后重新打包。

## 常见问题

- **安装时如何更改安装路径？**
  在安装程序界面中，可以点击"浏览"按钮选择自定义安装位置

- **安装后找不到应用？**
  查看开始菜单中的"Electron Paper"文件夹

- **打包过程中报错？**
  确保没有应用实例在运行，关闭所有相关进程后重试

- **无法删除旧版本？**
  手动结束所有"Electron Paper.exe"进程后再尝试打包
  
- **翻译功能无法使用？**
  确保已正确配置SiliconFlow API密钥，并选择了合适的翻译模型

- **链接无法打开？**
  应用使用系统默认浏览器打开外部链接，确保系统已正确配置默认浏览器

- **AI助手回复格式问题？**
  如果遇到分隔线或快捷检索选项显示异常，可尝试重启应用解决

## 可视化助手项目化与持久化方案（最终目标）

### 最终目标

- 用户在可视化助手中完成数据上传、图表生成、微调等操作后，可以将整个项目保存为结构化的JSON文件，实现完整的项目持久化。
- 项目结构化存储内容包括：
  1. **项目名称**：用户自定义的项目名。
  2. **上传数据**：建议将原始CSV文件内容转为base64字符串后包裹在JSON中，或直接存储为csv文件并在JSON中记录文件名/路径。
  3. **生成区对话历史**：保存用户与生成agent的所有对话消息（含问题、代码、结果等）。
  4. **画布内容**：保存所有图表对象的结构化信息，包括：
     - 图表类型、位置、尺寸、选中状态等
     - 生成的python绘图代码
     - 图片内容（推荐：仅在前端渲染时用base64，持久化时可选base64或本地图片路径，或图片hash/摘要）
  5. **图表微调区历史**：保存用户与微调助手的所有对话消息。
  6. **其它元数据**：如创建时间、最后修改时间、项目描述等。

### 内存优化与前后端解耦新方案

- 图表生成和微调阶段，后端直接在内存中暂存PNG图片对象，并与对应的python绘图代码绑定，前端无需base64传递，直接通过内存对象渲染画布。
- 前后端通过内存对象和结构化数据（如python代码、图片对象引用）实现完全解耦，无需任何磁盘中转或base64转换，极大提升性能和交互体验。
- 只有在用户选择"导出项目"时，才会将所有界面内容（包括上传的数据文件、画布内的PNG图片等）一并保存到项目文件夹内，并生成结构化JSON索引文件。
- 数据文件（如csv）会被复制到项目文件夹，json索引中的数据文件路径应指向项目内的相对路径，避免原始数据文件路径变更导致项目不可用。
- JSON索引文件包含：
  - 项目基本信息（名称、描述、创建/修改时间等）
  - 数据文件索引（如csv文件名或路径）
  - 生成区和微调区的聊天历史
  - 画布内所有图片的类型、加载路径、图片位置、图片大小、绑定的python代码等结构化信息
- 这样既保证了前后端解耦和高性能，也便于后续项目的导入、迁移和复现。

### 方案总结

- **渲染时用base64，持久化时可选base64或文件路径**，兼顾性能与便携性。
- **数据、图片分离存储，JSON只做结构化索引**，便于大项目管理。
- **支持zip打包导入导出，便于用户迁移和备份。**
- **所有链路和历史均结构化存储，方便后续AI微调、复现和追溯。**

### 项目导入导出说明

- 导出项目时，系统会自动创建一个项目文件夹，结构如下：
  ```
  MyProject/
    ├── project.json         # 项目索引文件（结构化描述所有内容）
    ├── data/                # 数据文件目录
    │   └── data.csv         # 上传的数据文件（可有多个）
    ├── images/              # 图片资源目录
    │   ├── chart1.png       # 画布内的图片
    │   └── chart2.png       # 更多图片
    └── ...                  # 其它资源
  ```
- 支持一键打包为zip文件（如"MyProject.zip"），便于迁移和分享。
- **导入项目时，仅支持以文件夹形式导入**，不支持直接导入zip包，需先解压为文件夹后再导入。
- 推荐优先使用zip包进行项目的备份、迁移和分享，保证所有资源完整、路径正确、跨平台无依赖。

### 项目导出交互说明

- 点击"导出项目"时，会弹出导出弹窗，包含以下内容：
  1. 选择导出路径。常用导出路径在"系统设置"标签页下保存，弹窗会自动读取该路径作为默认值；若系统设置中未保存路径，则显示为空，要求用户手动选择。
  2. 填写项目名称，默认取画布区上方的"project title"，用户可在弹窗内直接修改。
  3. 选择导出格式，格式选择器默认为文件夹，第二选项为ZIP。
  4. 填写项目描述（非必填项）。
- 用户确认后，系统将项目内容导出到指定路径下的文件夹或zip包。
